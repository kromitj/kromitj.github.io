Version Control

WHY YOU SHOULD BE USING IT

01/03/2016

Let's Figure Out What It Is.

Version control is a software system that tracks the changes made to a project and keeps a archivable back-log of the changes; It's like a high-tech, non-volitile, undo button. Version control allows you to go back in time and see the source of your project at that any particular time. This is helpful, because if you majorly screw-up your program one day and can't find the source of the error, you can use version control to revert to a previous stable version on the program.

Let Talk About Git

Git was a creation of neccessity. The creaters of Git were people who were developing for the open source project of Linux. They had been using another kind of version control called, BitKeeper. But the relationship between the company that owns BitKeeper and the Linux developers fell through, so the Linux developers began working on their own version of version control; most credit is given to Linus Torvalds, who also created Linux. Their requirments for this new VCS included:

  • Be Speedy
  • Simple Design
  • Strong Support for non-linear development(meaning bunches of parallel branches)
  • Fully distributed(Whatever that means)
  • Able to handle Large projects while still being responive and quick

Let Talk About GitHub

GitHub is a website that functions as a web-based git repository. Where Git is only alive locally on computers, GitHub allows you to store your git repositories on the web. That's pretty Neat! This comes in handy because say, if you spill a gallon of milk on your laptop and loss everything, you will still have the files hosted on GitHub. It also makes working with a team of people easier because it allows for people to contribute to the project no matter where they are and it keeps everybody on the same page by keeping master copy of the project and making sure that people aren't steping on each others toes. One other plus to using GitHub is you can display amazing code to the world, including potential employers.

So What Did We Learn?

  • Version Control is a way to take snap-shoots of your project progressivly in time, and allows you to go back to those snap-shootsd if you need to
  • Break your Code? No Big deal, just go back in time
  • Git is just a Version Control System(VCS)
  • GitHub is a web-based git repository
  • GitHub is great for teams, because it keeps everyone up-to-date and in-check
  • GitHub functions as a remote repository so if your computer crashes your code is still save
  • GitHub is used as a portfolio for coders
