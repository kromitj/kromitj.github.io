A Little History on Object-Orientated Programming

Ruby is a Object-Orientated Language, a paradigm that came into it's own in the 1980's with the growth of C++; there were many O.O. languages before, but C++ took the wildly popular C language and gave a wrapper around it that allowed it to function in a OOP manner. The underlining C'ness of C++ allowed it to be adapted quickly. Before the influx of OOP, programmers followed the Procedural paradigm of programming, which organized instructions into groups that perform one task; this one task is called a procedure. The procedure was a named method that included a structured block of instructions, that would be executed every time the program called the procedure by name. This was all gravy, but programmers began to wish that programs could more accurately and easily mimic the real world. It was defined that the world is made up of objects and every object has a certain amount of attributes and behavior; this is the bread and butter of the OOP paradigm. It is coming from a place were we structure the program into objects, where each object has its own unique abilities and attributes; this grouping of behavior and properties is called, encapsulation. In other words, encapsulation is forming a new object by grouping together related things, like a key-chain groups together a bunch of keys, which are all have similar behavior but have different properties. In OOP, encapsulation is grouping together attributes and behaviors to form a object.

Enough Already, Let's Make A Class

Alright, well we're going to create a class encapsulates the the attributes and behaviors of South Park Characters. Let's see the line up :

So the Class name is going to be Character and the first thing we are going to do is define important attributes and behaviors that all the characters have:

Attributes:
  • First Name: String
  • Last Name: String
  • Wears Hat?: Bool
  • Shirt Color: String
Behaviors:
  • Say Curse Word
  • Fake Butters Death
  • Play Guitar Hero

Now we will write the class definition and create an instance object of the class:


class Character
  def initialize(first, last, has_hat, shirt_color)
    @first_name = first
    @last_name = last
    @wears_hat = has_hat
    @shirt_color = shirt_color
  end
end # end of class definition

kyle = Character.new("Kyle", "Broflovski", true, "Orange")

Character:0x007f8fb0ab0f08 @first_name="Kyle", @last_name="Broflovski", @wears_hat=true, @shirt_color="Orange" # terminal output

What is happening in the above code is we are defining a class with a name of Character, then inside of the class we are defining a method, it is named "initialize", which is a method that is inherited from the Object class, most if not all classes automatically inherit from this class. But anyway, the initialize method takes 4 arguments, and then assigns their values to Class' instance variables, which are the same thing as attributes. Note: the "@" in-front of the instance variables just defines them to be an instance variable. With the initialize method defined you can now create your very own instance objects of the class; that is what the local variable named kyle is being assigned to. If your wondering why we didn't call the "initialize" method when created the object, it's because the "new" method is actually behind the scenes calls "initialize"; confusing, I know! The final line just shows what the terminal outputs as the Character object.

Now, let's add some functionality to the Character class. In the specs, we described that we wanted a functionality of the Character class to be curing, so lets give them this much needed skill:


class Character
  def initialize(first, last, has_hat, shirt_color)
    @first_name = first
    @last_name = last
    @wears_hat = has_hat
    @shirt_color = shirt_color
  end
  # new method will be created here...
  def say_curse
    puts "God Damn-it, Kyle!!!"
  end
end # end of class definition

cartman = Character.new("Eric", "Cartman", true, "Red")
cartman.say_curse
God Damn-it, Kyle!!! # termainal output

So this shows us adding functionality to the Character class; the functionality of being able swearing! I would like you to add the other behaviors we outlined in the spec portion and also add a few new attributes like, hat or glove color.
