Are You Saying KFC Buckets Are Arrays?

Kinda, yeah! An array is defined as a ordered, integer-indexed collection of objects. A KFC bucket is a collection of chicken were its index is the order your grubby hands take the chicken out of the bucket; one drumstick, "num num num", two drumstick, "num num num", "uh I hate myself for eating this garbage", sign, three drumstick, "num num uhhhh nu...". As you work your way through the bucket the length of the KFC array also begins to strink, but what if I told you that the last element of the KFC Array actually has an index of 6. This happens because in Computer Science the first element of something starts at zero, opposed to starting at one; whoa, that's mind-bendingly trippy, man!" This stift in logic causes what is known as a fencepost error or an off-by-one error.

my_chicken = ['1st piece', 'first piece', '3rd piece', '4th piece', '5th' piece', '6th piece', '7th piece'] my_chicken[0] # => 1st piece my_chicken[6] # => 7th piece my_chicken[7] # => nil

Here you can see that the my chicken array at index zero is the first element, which has a value of '1st piece'. The last element is at index six, with a value of '7th piece'. Finally even thought the array has seven elements if you try to access the 7th index of the my_chicken array you will get a return value of nil.

Ok, so lets see how using the programming language ruby we can create an array. Here are your options:

my_first_array = [1, 2, 3, 4, 5, 6] # this creates an array with 6 elements of type:Integer my_second_array = ['a','e','i','o','u'] # this creates an array with 5 elements of type:String kitchen_sink_array = ["Sink", 1, 13.5, kitchen_object] # four element array with all differant object types empty_array = Array.new # => [] this creates an emty array three_nil_array = Array.new(3) # creates a three element array, by default they all get set to nil all_at_zero = Array.new(5, 0) # creates a five element array with initial values set to 0

#LetsLearnAboutHashes #NotTheDrug #NotLikeHashbrowns

Hashes are similar to Arrays. Where the Hashes differ is instead of using and integer index, a Hash uses a key. The hash consists of key/value pairs were the key is a unique object that differentiates itself from all its sibling Hashes. Here is an example of what hashes look like:

artist_best_song = {"Eminem" => "The Way I AM", "Elvis Costello" => "(I Don't Want To Go To) Chelsea", "Justin Bieber" => "Baby, Baby"}

You can also use symbols for the key instead of string, like this:

car_ratings = { :civic => 4.5, :300m => 1.5, :911 => 5, :mini-cooper => 4.5} car_ratings = { civic: 4.5, 300m: 1.5, 911: 5, mini-cooper: 4.5}

Both of them create the exact same Hash with the same symbol keys. You can create an empty array by using the class method 'new', like this:

empty_hash = Hash.new # => {}

One thing that I find harder with Hashes is to isolate specific values or specific keys. To to get the value of from a hash element you can put in the key name into the index of the created hash, like this:

car_ratings[:mini-cooper] # => 4.5

Hip Hip Array We're Done!

    Things That We Learned:
    • Arrays or collections of data with integer indexs
    • Arrays can be filled all sorts of objects
    • You select an element of an Array by its index,like this: my_array[1]
    • Arrays start and index 0, watch-out for fence-post errors!
    • Theres more then one way to create Arrays and Hashes
    • Hashes are collections of key:value pairs
    • Each key in a Hash has to be unique
    • The prefered value for a key is a symbol(:symbol)
    • You select an element of an Hash by indexing its key, like this: car_ratings[:civic]
